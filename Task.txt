Overview
Speed is a game in which the player seeks to type as many words they see to earn points.



Rules
Speed is played according to the following rules.

The game begins with five words moving on the screen. <-Actors (word)
The player tries to type the words they see on the screen. <-Player
The player earns points for the words they type correctly. <-word.getPoints
If the player presses the enter key their buffer is cleared.
Play continues until the player presses the ESC key.



Requirements
Your program must also meet the following requirements.

The program must use the Speed template.
The program must have a README file with assignment and author names.
The program must have at least eight classes.
Each module, class and method must have a corresponding comment using the style demonstrated in the solo checkpoint.
The game must remain generally true to the order of play described in the rules.



Make the game your own by enhancing it any way you like. A few ideas are as follows.

Enhanced words (different list, more on screen, etc)
Enhanced input validation with user-friendly messages.
Enhanced game play (variable speeds, expire penalties, etc).
Enhanced board representation (better ascii art)
